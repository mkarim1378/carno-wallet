# نظام کیف پول پیشرفته - راهنمای استفاده

## 📋 درباره این ویژگی

این افزونه قابلیت استفاده از اعتبار کیف پول کاربر را برای پرداخت خریدها فراهم می‌کند. سیستم دو حالت پرداخت را پشتیبانی می‌کند:

### حالت ۱: پرداخت کامل از کیف پول
اگر موجودی کیف پول **بیشتر یا برابر** با مبلغ سفارش باشد:
- ✓ کل مبلغ سفارش از کیف پول کم می‌شود
- ✓ اعتبار باقی‌مانده در کیف پول می‌ماند
- ✓ سفارش بلافاصله کامل می‌شود

### حالت ۲: پرداخت جزئی (کیف پول + درگاه پرداخت)
اگر موجودی کیف پول **کمتر از** مبلغ سفارش باشد:
- ✓ کل موجودی کیف پول کم می‌شود
- ✓ باقیمانده مبلغ از طریق درگاه پرداخت دریافت می‌شود
- ✓ سفارش در انتظار پرداخت درگاه باقی می‌ماند

---

## 👤 تجربه کاربری

### در سبد خرید:
1. کاربر وارد شده و موجودی کیف پول داشته باشد
2. یک کادر آبی رنگ نمایش داده می‌شود که:
   - موجودی فعلی کیف پول را نشان می‌دهد
   - دکمه "استفاده از کیف پول" یا "لغو استفاده" را نمایش می‌دهد
3. کاربر می‌تواند با کلیک بر دکمه، از کیف پول استفاده کند
4. بلافاصله مبلغ کیف پول از قیمت کل کم می‌شود

### در صفحه تسویه‌حساب:
1. خلاصه استفاده از کیف پول نمایش داده می‌شود:
   - مبلغ کل سفارش
   - مبلغ کم‌شده از کیف پول
   - مبلغ باقی‌مانده (در صورت وجود)
2. اگر مبلغ کاملاً از کیف پول تأمین شود:
   - پیام "✓ سفارش شما کاملاً از طریق کیف پول پرداخت می‌شود" نمایش داده می‌شود
3. اگر پرداخت جزئی باشد:
   - کاربر می‌تواند درگاه پرداخت دلخواه را انتخاب کند

## 🔧 جزئیات تکنیکی

### فایل‌های جدید:
- `includes/class-wallet-cart.php` - مدیریت کیف پول در سبد خرید
- `assets/wallet-cart.js` - جاوا اسکریپت برای تعامل کاربر

### تغییرات فایل‌های موجود:
- `carno-wallet.php` - اضافه‌کردن بارگذاری کلاس جدید
- `class-wallet-core.php` - اضافه‌کردن هندلینگ بازپرداخت
- `class-wallet-gateway.php` - پشتیبانی برای پرداخت جزئی
- `admin-style.css` - اضافه‌کردن سبک‌های جدید

### داده‌های Meta سفارش:
- `_carno_wallet_used` - بیان کننده استفاده از کیف پول
- `_carno_wallet_amount` - مبلغ کم‌شده از کیف پول
- `_carno_wallet_partial_payment` - نشان دهنده پرداخت جزئی
- `_carno_wallet_amount_remaining` - مبلغ باقی‌مانده برای پرداخت
- `_carno_wallet_deducted` - نشان‌دهنده اینکه موجودی کم‌شده یا نه
- `_carno_wallet_refunded` - نشان‌دهنده بازپرداخت به کیف پول

### Session Keys:
- `carno_wallet_credit_used` - فعال/غیرفعال بودن کیف پول در سشن فعلی
- `carno_wallet_deduct_amount` - مبلغ کم‌شده از اعتبار

## 🛠️ نکات مهم برای توسعه‌دهندگان

### AJAX Endpoints:
```javascript
// فعال کردن کیف پول
wp_ajax_carno_apply_wallet_credit

// غیرفعال کردن کیف پول
wp_ajax_carno_remove_wallet_credit
```

### Hooks:
```php
// اعمال خصم به قیمت
woocommerce_cart_calculate_fees

// ذخیره اطلاعات در سفارش
woocommerce_checkout_create_order

// پس از تغییر وضعیت سفارش
woocommerce_order_status_changed

// هنگام بازپرداخت
woocommerce_refund_created
```

### کلاس‌های کمکی:
```php
// بررسی اعمال کیف پول
Carno_Wallet_Cart::is_wallet_credit_applied()

// دریافت مبلغ کم‌شده
Carno_Wallet_Cart::get_deducted_amount($order_id)

// بررسی استفاده از کیف پول در سفارش
Carno_Wallet_Cart::order_used_wallet($order_id)

// اضافه کردن موجودی (برای بازپرداخت)
Carno_Wallet_Core::add_to_balance($user_id, $amount)

// کم کردن موجودی
Carno_Wallet_Core::deduct_balance($user_id, $amount)
```

## 🚀 مثال‌های کاربردی

### مثال ۱: کاربر با موجودی 500,000 و سفارش 300,000 تومانی
```
موجودی کیف پول: 500,000 تومان
مبلغ سفارش: 300,000 تومان
نتیجه: کل مبلغ از کیف پول کم می‌شود
موجودی نهایی: 200,000 تومان
```

### مثال ۲: کاربر با موجودی 200,000 و سفارش 500,000 تومانی
```
موجودی کیف پول: 200,000 تومان
مبلغ سفارش: 500,000 تومان
نتیجه: 200,000 تومان از کیف پول کم می‌شود
مبلغ برای درگاه: 300,000 تومان
موجودی نهایی: 0 تومان
```

## ⚠️ مواردی که باید در نظر بگیرید

1. **بازپرداخت‌ها**: هنگام بازپرداخت تمام یا بخشی از سفارش، اعتبار به کیف پول برگردانده می‌شود
2. **حذف کردن**: اگر کاربر کیف پول را منتخب کرده ولی گیت‌وی دیگری را انتخاب کند، سفارش هنوز از کیف پول کسر می‌شود
3. **تعدیل سفارش**: اگر کاربر سفارش را تحریر کند، موجودی کیف پول تعدیل شود

## 🐛 رفع مسائل رایج

### مسئله: کیف پول به سفارش اضافه نمی‌شود
**حل**: بررسی کنید که:
- کاربر وارد شده باشد
- موجودی کیف پول > 0 باشد
- AJAX به درستی کار کند (نونس را بررسی کنید)

### مسئله: مبلغ اشتباه کم می‌شود
**حل**: بررسی کنید که:
- هیچ درگاه دیگری مبلغ را دو بار کم نکند
- WooCommerce Cache تخلیه شود

### مسئله: موجودی بعد از پرداخت تصحیح نمی‌شود
**حل**:
- وضعیت سفارش را به "processing" یا "completed" تغییر دهید
- عملیات خودکار در `woocommerce_order_status_changed` اجرا می‌شود

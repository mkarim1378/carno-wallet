# راهنمای نصب و راه‌اندازی کیف پول

## 📦 فایل‌های نصب‌شده

بعد از اجرای این آپدیت، فایل‌های زیر اضافه/تغییر کرده‌اند:

### ✅ فایل‌های جدید:
- `includes/class-wallet-cart.php` - مدیریت کیف پول در سبد خرید و تسویه‌حساب
- `assets/wallet-cart.js` - جاوا اسکریپت برای تعامل کاربر
- `WALLET_FEATURE.md` - مستندات کامل ویژگی

### 📝 فایل‌های تغییر‌یافته:
- `carno-wallet.php` - اضافه‌کردن بارگذاری کلاس `Carno_Wallet_Cart`
- `includes/class-wallet-core.php` - اضافه‌کردن `handle_refund()` و `add_to_balance()`
- `includes/class-wallet-gateway.php` - تغییر `is_available()` و `process_payment()` برای پشتیبانی از پرداخت جزئی

```bash
# بررسی اینکه WooCommerce نصب و فعال است

```

## ⚙️ تنظیمات

### تنظیمات WooCommerce برای Wallet Gateway
1. به **WooCommerce** → **تنظیمات** → **پرداخت** بروید
2. از **درگاه‌های پرداخت** بخش **کیف پول** را فعال کنید
3. عنوان و توضیح را حسب نیاز تغییر دهید

### تنظیمات لازم در wp-config.php (اختیاری)
```php
// برای Debug:
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```

## 🔍 مشاهدات عملکردی

### لاگ‌های مهم
```
یادداشت‌های سفارش ← بررسی اینکه موجودی به درستی کم شده
Order Meta ← بررسی `_carno_wallet_amount` و `_carno_wallet_used`
Session ← بررسی `carno_wallet_credit_used` و `carno_wallet_deduct_amount`
```

### Database Queries
```sql
-- مشاهده موجودی کاربر
SELECT user_id, meta_key, meta_value 
FROM wp_usermeta 
WHERE meta_key = 'user_wallet_balance'
LIMIT 10;

-- مشاهدة سفارش‌های که از کیف پول استفاده کردند
SELECT post_id, meta_key, meta_value 
FROM wp_postmeta 
WHERE meta_key LIKE '_carno_wallet%'
LIMIT 10;
```

## 🎯 سناریوهای تست

### سناریو ۱: خرید کامل از کیف پول
```
موجودی کاربر: 500,000 تومان
قیمت محصول: 300,000 تومان

مراحل:
1. محصول به سبد اضافه کنید
2. "استفاده از کیف پول" را فشار دهید
3. قیمت کل = 0 تومان (یا برعکس عرض‌شده)
4. به درگاه "کیف پول" برای پرداخت بروید
5. سفارش کاملاً تکمیل شود
6. موجودی = 200,000 تومان (500,000 - 300,000)
```

### سناریو ۲: خرید جزئی + درگاه پرداخت
```
موجودی کاربر: 200,000 تومان
قیمت محصول: 500,000 تومان

مراحل:
1. محصول به سبد اضافه کنید
2. "استفاده از کیف پول" را فشار دهید
3. قیمت نمایش‌شده = 300,000 تومان (500,000 - 200,000)
4. درگاه دیگری را برای 300,000 تومان انتخاب کنید
5. پرداخت کنید
6. موجودی = 0 تومان (200,000 - 200,000)
```

### سناریو ۳: لغو و تغییر مسیر
```
موجودی کاربر: 300,000 تومان
قیمت محصول: 500,000 تومان

مراحل:
1. محصول به سبد اضافه کنید
2. "استفاده از کیف پول" را فشار دهید (400,000 نمایش)
3. "لغو استفاده" را فشار دهید
4. قیمت کل باید دوباره 500,000 تومان باشد
5. تنها درگاه‌های دیگر قابل‌دسترسی باشند
```

### سناریو ۴: بازپرداخت
```
سفارش از کیف پول: 200,000 تومان (موجودی قبل: 500,000)
موجودی بعد: 300,000 تومان

بازپرداخت 100,000 تومان از 200,000:
→ موجودی نهایی: 400,000 تومان

بازپرداخت تمام (200,000 تومان):
→ موجودی نهایی: 500,000 تومان (بازگشت به حالت اول)
```

## 📞 خطای رایج و حل‌ها

| خطا | علت | حل |
|------|------|-----|
| کیف پول در سبد نمایش نمی‌شود | کاربر وارد نشده یا موجودی ندارد | کاربر را بررسی کنید |
| مبلغ اشتباه | Cache فعال است | WooCommerce Cart Cache را پاک کنید |
| موجودی بروزرسانی نمی‌شود | Order status تغییر نکرده | Status را به "processing" تغییر دهید |
| AJAX خطا | Nonce غلط | صفحه را رفرش کنید |
